# Contract: Generated Service Layer Dependencies

## Scope

Defines allowed project-reference directions for services generated by `nfw add service`.

## Layer Set

- `Domain`
- `Application`
- `Infrastructure`
- `Api`

## Allowed References

- `Domain` -> *(none within service layer set)*
- `Application` -> `Domain`
- `Infrastructure` -> `Application`, `Domain`
- `Api` -> `Application`, `Infrastructure`

## Forbidden References

- `Domain` -> `Application`, `Infrastructure`, `Api`
- `Application` -> `Infrastructure`, `Api`
- `Infrastructure` -> `Api`
- `Api` -> `Domain`

## Verification Expectations

- Generated project references satisfy allowed set immediately after generation.
- Integration tests assert no forbidden edge exists.
- Template changes that break this contract are treated as regression failures.
